✅ **Proper Timestamp Generation**
- Format: YYYYMMDDHHmmss
- Auto-generated by system

✅ **Password Encryption**

### Production Setup

```env
NODE_ENV=production
MPESA_SANDBOX=false
MPESA_CONSUMER_KEY=<production_key>
MPESA_CONSUMER_SECRET=<production_secret>
MPESA_SHORTCODE=<your_shortcode>
MPESA_PASSKEY=<your_passkey>
MPESA_CALLBACK_URL=https://jkuat-queue.youromain.com/api/queue/mpesa-callback
```

### Callback URL Setup

Your callback URL must be:
- ✅ Publicly accessible (not localhost)
- ✅ Using HTTPS (SSL certificate required)
- ✅ Responding with HTTP 200
- ✅ Registered in M-PESA account settings

## 🐛 Troubleshooting

### Payment Not Sending

**Error**: "Invalid Consumer Key/Secret"
- ✓ Verify credentials in `.env`
- ✓ Copy-paste from Daraja (avoid typing errors)
- ✓ Check for extra spaces/newlines

**Error**: "Invalid timestamp"
- ✓ System auto-generates, should be automatic
- ✓ Check server time is synced
- ✓ Verify format: YYYYMMDDHHmmss

**Error**: "Phone number validation failed"
- ✓ Must be 254XXXXXXXXX format
- ✓ Must be 12 digits
- ✓ No special characters

### Callback Not Received

- ✓ Callback URL must be public (not localhost)
- ✓ Must use HTTPS with valid SSL
- ✓ Check firewall allows inbound HTTPS
- ✓ Verify URL in M-PESA settings matches
